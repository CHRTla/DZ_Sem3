# Конспект семинара_2 от 20.12.22
В Git по- умолчанию основная ветка именнуется в *master*. Эта ветка создается автоматически командой *git init*. Прижелании, её можно переименовать. Эта ветка равнозначна другим веткам,которые будут созданы в процессе работы.Сегодня мы будем использовать ветку *master* в качестве **чистовика**, в остальных ветках мы будем вносить изменения, как в **черновиках**

HEAD- это указатель на текущую локальную ветку.

Совет: поставьте режим *auto save*, чтобы не забывать сохранять изменения.

__Создание веток в локальном репозитории__

Для создания ветки используется команда *git branch new_branch_name*. Чтобы начать работу в новой ветке, необходимо перейти в неё командой *git checkout* . Затем вносим необходимые данные или изменения в вашем проекте. После завершения всегда выполняем следующие действия:
* сохраняем данные *Save* или сочетанием клавиш *Ctrl+S* (если по умолчанию не указано *auto save*)
* выполняем команду **_git add .имя файла с расширением_** (происходит отслеживание файла)
* фиксируем наши изменения командой  **_git commit -m "некий коментарий по проделанной операции"_**

Таким способом создаются следующие ветки. Сегодня мы будем создавать 4-е ветки.


__Слияние веток__

Прежде чем приступить к слиянию веток, необходимо вызвать команду *git branch*, чтобы посмотреть в какой ветке вы находитесь. Cимвол (*) указывает на актуальную ветку. 

Корректное слияние осуществляется только в ветке *master*. Поэтому  для начала,необходимо перейти в основную ветку коммандой **_git checkout master_**.

Слияние осуществляется командой **_git merch имя ветки_**

Снова выполняем команды:
* Save
* git add .
* git commit -m ""

Варианты **удачного слияния**:
1. _fast-forward_ происходит, когда основная ветка "догнала" побочную ветку. Изменение произошло только в побочной ветке.

2. _ort stratage_ происходит, когда изменения происходят в обеих ветках без конфликта

__Конфликт веток__

Конфликты возникают, когда затрагивают общее рабочее пространство. 
Способы разрешения конфликта:

1. Вручную убрать символы <<< перед HEAD и >>>>> перед Conflict
2. Из указанных вариантов выбрать оптимальный:
 * Accept Current Change - ставим главнее основную ветвь HEAD 
 * Accept Incoming Change - ставим главнее побочную ветвь
 * Accept Change both - принять оба изменения







Конфликт разрешится после последовательности выполнения команд:



* save
* git add .
* git commit -m ""

__Создание удаленного репозитория в GitHub__
1. Регистрируемся на сайте *GitHub.com* 
2. Создаем ваш новый удаленный репозиторий. Это можно сделать одним из нескольких способов:
* в главном окне выбираем команду *start a new Recository*
*нажать правой кнопкой на иконку вашего аватара и выбрать *yours repositories*- *new*
* возле иконки вашего аккаунта нажимаем значок (+) -*new repository*

3. Откроется новое окно. В окне ( справа от окна *owner* с вашим именем пользователя) вносим уникальное имя репозитория
4. Выбираем ссылочный тип *HTTPS* 
5. Выбираем досуп *Public* (на время обучения)
6. Нажимаем на *Create Repository* и переходим на новую страницу.
7. Здесь вам предложат 3 варианта:

     1. создать новый репозиторий "с нуля"
     2. связать на основе существующего репозитория
     3. или импортировать уже готовый

    Выбираем 1 пункт
8. Переходим  на ПК   

__Синхронизация локального и удаленного репозитория__

1. На рабочем столе ПК создаем новую папку с таким же названием как на GitHub (желательно)
2. Открываем папку через программу *VisualStudioCode (VSCode)* 
3. Внутри этой папки создаем новый файл README.md (заглавными буквами с расширением). Появится информационный файл с сиволом в кружке (i).
4. Запускаем *terminal* и Выполняем последовательно комманды, прописанные на **GitHub*, а именно:
 * git init
 * git add .\README.md
 * git commit -m "ваш комментарий"
 * git branch -M master (ВАЖНО!: **_M_**- заглавная, а ветка должна иметь одно название в локальном и удаленном репозитории).
 * git remote add origin http:\\... (ВАЖНО!: Вставляем эту команду копированием с сайта)
 * git push -u master (создаем связь с нашей веткой).
5. Обновить страницу GitHub
Мы связали (синхронизировали) наш локальный и удаленный репозиторий. Поздравляю!
  

__Внесение изменений из локального репозитория на удаленный репозиторий GitHub__

1. Открываем нашу папку, созданную ранее, в программе VSCode и вносим необходимые изменения.
2. Далее мспользуем стандартную последовательность команд:
* save 
* git add .\README.md
* git commit -m "ваш комментарий"
3. Смотрим журнал, используя команду:
* *git log* 

  (видим, что основная ветка локального репозитория **_HEAD_** ушла вперед, а ветка с удаленного репозитория **_origin/master_** осталась позади).

4. Поэтому нам нужно связать эти ветки командой:
* git push 

Вся проделанная работа в локальном репозитории, будет отражаться в удаленном. 

__Внесение изменений из удаленного репозитория GitHub на локальный репозиторий__

1. На сайте GitHub войти в свой репозиторий и нажать на файл README 
2. Выбрать редактирование (значок карандаша)
3. В кодовой версии внести изменения
4. Добавить комментарий в *commit changes* 
5. Выбрать * commit directly to the master branch (Если вы работаете в одиночку над своим проектом) 

   Если работаете в команде, где необходимо согласовывать проект с руководством, то выбираете *create a new  brance this commit start a pull request* 
6. Нажимаем кнопку **Create change**
7. Переходим в VSCode и в терминале даем команду:

   ВАЖНО!: Перед слиянием ,для подстраховки, команду:

   * **_git push_**
   
    (она сохранит ваши изменения, если вы случайно забыли)

  * git pull ( команда для загрузки изменений из удаленного в локальный репозиторий)

  
__Клонирование репозитория через команду *fork*__ 

1. Перейти по ссылке, которую вы получили в GitHub
2. Возле вашей аваталки нажать на *fork* (вилка)
3. Видим, что название репозитория напротив вашего имени пользователя
4. Нажимаем **create fork** 
Мы создали fork версию на своем репозитории 
5. Проверяем, чтобы основная ветка совпадала с вашей (*master*)
6. Нажимаем **CODE** и копируем ссылку. 

    Именно эта ссылка создает связь между вашим репозиторием и первоисточником) 
8. Выбираем или создаем папку на ПК
7. Открываем VSCode, добавляем папку или открываем папку. Через  терминал задаем команду :

 * git clone ссылку вставить через пробел

8. Переходим в клонированную папку ивыполняем команду перехода:
  * cd .\ (через **Tab** выбрать клонированную папку)
 9. Создаем побочную ветку с вашей фамилией и даём команду:
   * git branch через пробел имя ветки (ваша фамилия)
10. Переходим в побочную ветку командой:

 * git checkout через пробел имя ветки

 Мы теперь можем работать в этой ветке. 

 11. После внесения изменений выполняем знакомую последовательность команд:
 * save
 * git add .
 * git commit -m "ваш коментарий"
 Для подстраховки запускаем команду:
 * git push ( она не должна сработать, т.к. нет связи )
 12. Теперь нужно создать связь с нашей веткой в удаленном репозитории и "толкнуть" изменения в него командой:

 * git push -u origin имя вашей ветки
 13. Обновляем GitHub и видим, что только что создали новую ветку. Сохраняем данные изменения. нажав на зеленую кнопку **compare & pull request**
 
14. В открывшемся окне видим свои изменения и теперь отправим изменения на проверку. для этого нажимаем **create pull request**
