# Основы работы с ***Git***
## Установка необходимого ПО
Для начала работы, необходимо установить следующие программы:
- ***Git*** скачать [тут](https://git-scm.com/book/en/v2/Getting-Started-Installing-Git) (https://git-scm.com/book/en/v2/Getting-Started-Installing-Git)
- ***VS Code*** скачать [тут](https://code.visualstudio.com/) (https://code.visualstudio.com/)

После этого, необходимо сначала установить ***Git***, и только после этого установить ***VS Code***
После установки необходимо «представиться» системе контроля версий. После установки необходимо «представиться» системе контроля версий. Это нужно сделать всего один раз, и git запомнит вас. Для этого нужно ввести в терминале 2 команды: 
```sh
git config --global user.name "Ваше имя английскими буквами"

```
## Основные команды Git:
### ***Git version*** - отображение информации о версии Git
```sh
git --version
```
### ***Git add*** - добавление содержимого рабочего каталога в индекс
Команда `git add` добавляет содержимое рабочего каталога в индекс (staging area) для последующего коммита.
```sh
git add filename.txt #добавление конкретного файла
git add ./ #добавление всех файлов в текущей папке
```
## ***Git status*** - отображение состояния файлов в рабочем каталоге
Команда `git status` показывает состояния файлов в рабочем каталоге и индексе: какие файлы изменены, но не добавлены в индекс; какие ожидают коммита в индексе.
```sh
git status
```
## ***Git diff*** - вычисление разницы между любыми двумя Git деревьями
Команда `git diff` используется для вычисления разницы между любыми двумя Git деревьями. Это может быть разница между вашей рабочей копией и индексом (собственно *git diff*), разница между индексом и последним коммитом (*git diff --staged*), или между любыми двумя коммитами (*git diff master branchB*).
```sh
git diff
```
## ***Git commit*** - фиксация и сохранение слепка внутри вашего репозитария
Команда `git commit`. Теперь, когда ваш индекс находится в таком состоянии, как вам и хотелось, вы можете зафиксировать свои изменения. Запомните, всё, что до сих пор не проиндексировано — любые файлы, созданные или изменённые вами, и для которых вы не выполнили git add после редактирования — не войдут в этот коммит.
```sh
git commit -m "Ваш комментарий"
```
## ***Git log*** - просмотр журнала изменений
Журнал изменений. Перед переключением версии файла в Git используйте команду `git log`, чтобы увидеть количество сохранений
```sh
git log
```
## ***Git checkout*** - переключение между версиями
Для работы нужно указать не только интересующий вас коммит, но и вернуться  в тот, где работаем, при помощи команды `git checkout master`.
```sh
git checkout 4376 #необходимо указать как минимум первые 4 символа commit
```
## Ветвление:
## Способы создания веток и переключения между ними
Команда `git branch` — главный инструмент для работы с ветвлением. С ее помощью можно добавлять новые ветки, перечислять и переименовывать существующие и удалять их.

### Вывод списка всех веток
```sh
git branch
```
### Создание новой ветки
```sh
git branch <branch>
```
### Удаление ветки
```sh
git branch -d <branch>
```
### Переименование ветки
```sh
git branch -m <branch>
```
### Перечислить все удаленные ветки
```sh
git branch -a
```
### Слияние веток
```sh
git merge <name of merged branch>
```
### Выгрузка локального репозитория в удаленный репозиторий
```sh
git push <name of merged branch>
```
### Загрузка из удаленного репозитория в локальный репозиторий
```sh
git pull <name of merged branch>
```
### Связать ветки удаленного и локального репозитория
```sh
git push -u origin <name of merged branch>
```

## Работа с картинками
Работать с картинками очень просто. 
Вставка изображений:
```sh
![тут должна быть картинка](images1.png)
```
![тут должна быть картинка](images1.png)

Изображение со ссылкой:
```sh
[![Название картинки](images1.png)](https://google.com)
```

[![Название картинки](images1.png)](https://google.com)

В среде разработчиков не принято отслеживать файлы картинок.  Отслеживаются только файлы кода. Для добавления файла images1.png в игнор, нужно создать текстовый файл `.gitignore` и вписать туда название файла с картинкой `images1.png`

После создания файла `.gitignore` его необходимо добавить в репозитарий и закоммитить, выполнив следующую команду:

```sh
git add .gitignore
git commit -m "добавили .gitignore"
```
